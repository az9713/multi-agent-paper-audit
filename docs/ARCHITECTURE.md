## Big Picture

### What Problem Does This Solve?

Reading scientific papers critically is hard and time-consuming. Reviewers often miss issues because they:
- Don't have time to check every calculation
- Miss relevant prior work
- Accept claims without verification
- Have cognitive biases

This tool automates the critical reading process using specialized AI agents.

### Core Idea

Instead of one AI trying to do everything, we use **five specialized agents**, each expert in one aspect:

### Why Multiple Agents?

| Single Agent | Multiple Agents |
|--------------|-----------------|
| Jack of all trades | Each agent is specialized |
| Can miss things | Multiple perspectives |
| One point of failure | Redundancy |
| Hard to improve | Easy to modify one agent |
| Biased one way | Balance of views |

## System Components

### Component 1: Claude Code CLI

**What it is:** The AI-powered command line tool that runs everything.

**What it does:**
- Reads skill files
- Follows instructions
- Calls tools (web fetch, file write, etc.)
- Manages the conversation

**You don't modify this** - it's provided by Anthropic.

### Component 2: Skill Files

**What they are:** Markdown files that tell Claude Code what to do.

**Location:** `.claude/skills/paper-audit/`

**Structure:**
```
SKILL.md              â† Main orchestrator
agent-a-*.md          â† Agent A instructions
agent-b-*.md          â† Agent B instructions
agent-c-*.md          â† Agent C instructions
agent-d-*.md          â† Agent D instructions
agent-e-*.md          â† Agent E instructions
```

### Component 3: Output System

**What it is:** The files generated by the audit.

**Location:** `output/{paper_id}/`

**Structure:**
```
README.md             â† User entry point
deconstruction.json   â† Extracted claims
math_audit.md         â† Agent B output
adversarial_review.md â† Agent C output
decision_memo.md      â† Agent E output
verification/         â† Agent D output
```

### Component 4: External Services

**arXiv:** Source of papers
- Abstract: `https://arxiv.org/abs/{id}`
- PDF: `https://arxiv.org/pdf/{id}.pdf`
- HTML: `https://arxiv.org/html/{id}v1`

**Web Search:** For finding contradicting papers (Agent C)

## Design Decisions

### Why Markdown for Skills?

1. **Human readable** - Easy to understand and modify
2. **Version controllable** - Works with Git
3. **No compilation** - Changes take effect immediately
4. **Flexible** - AI adapts to context

### Why Parallel Agents B, C, D?

1. **Independence** - They don't depend on each other
2. **Speed** - Can run simultaneously
3. **Specialization** - Each focuses on one thing

### Why README.md as Entry Point?

1. **Discoverability** - Users naturally look for README
2. **Guidance** - Tells users where to start
3. **Overview** - Summarizes without overwhelming

### Why JSON for Claims?

1. **Structured** - Easy to process programmatically
2. **Extensible** - Can add new fields
3. **Portable** - Works with any programming language

### Why Python for Verification?

1. **Scientific computing** - NumPy, pandas, matplotlib
2. **Accessibility** - Most researchers know it
3. **Readable** - Easy to understand and modify
